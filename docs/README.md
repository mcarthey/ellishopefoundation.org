## ?? Documentation Standards

When adding new documentation:

1. **Location**: Place in appropriate subfolder
2. **Naming**: Use lowercase with hyphens (e.g., `setup-guide.md`)
3. **Format**: Use Markdown with clear headings
4. **Links**: Update this index when adding new docs
5. **Images**: Store in `docs/images/` folder
